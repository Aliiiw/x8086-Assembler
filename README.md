# x8086 Assembler

A small Python assembler for a limited subset of x86/8086-style instructions. The script reads instructions from `input.txt`, encodes supported register and memory forms, and prints the generated byte pattern as escaped hex bytes.

This is a course-style encoding project. It focuses on a narrow set of opcodes and addressing patterns rather than a complete x86 assembler.

## Features

- Reads multiple assembly instructions from `input.txt`
- Supports register-to-register operations
- Supports simplified memory/register operations with bracket syntax
- Handles 8-bit, 16-bit, and 32-bit register groups
- Emits output in escaped byte format, such as `\x66\x01\xd8`
- Uses only the Python standard library

## Supported Instructions

The assembler currently supports:

```text
add
or
and
sub
```

## Supported Registers

### 32-bit

```text
eax ecx edx ebx esp esi edi ebp
```

### 16-bit

```text
ax cx dx bx sp bp si di
```

### 8-bit

```text
al cl dl bl ah ch dh bh
```

## Input Format

Each line in `input.txt` should contain one instruction and two comma-separated operands:

```text
command operand1,operand2
```

Examples:

```text
add ax,bx
and al,bl
or ecx,ebx
sub [al],bl
add ax,cx
or cx,[bx]
```

Supported operand patterns:

```text
command register32,register32
command register16,register16
command register8,register8

command [register32],register32
command register32,[register32]
command [register16],register16
command register16,[register16]
command [register8],register8
command register8,[register8]
```

Registers are written without `%` or `$` prefixes. Memory operands use square brackets.

## How to Run

Install Python 3, edit `input.txt`, then run:

```bash
python3 Assembler.py
```

Example output from the included sample:

```text
\x66\x01\xd8\x20\xd8\x09\xd9\x28\x18\x66\x01\xc8\x66\x0b\x0b
```

## How It Works

`Assembler.py` contains:

- register lookup tables for 8-bit, 16-bit, and 32-bit registers
- opcode tables for register and memory forms
- `evaluateMod(status)` for ModR/M mode bits
- `isRegister(...)` for register-to-register encoding
- `isMemory(...)` for simplified memory/register encoding
- `printPatter(...)` for formatting opcode and ModR/M output as escaped hex bytes

The script reads `input.txt`, parses each line by splitting on spaces and commas, chooses register or memory encoding based on brackets, and appends each encoded instruction to one output string.

## Notes

- This is not a full x86 assembler.
- Only `add`, `or`, `and`, and `sub` are supported.
- The parser expects the exact input format shown above.
- Immediates, labels, jumps, comments, directives, and complex addressing modes are not supported.
- Memory operands are simplified to `[register]`; displacement, scale, index, and base combinations are not handled.
- Output is printed to the terminal and is not written to a file.
- 16-bit operations are emitted with the `\x66` operand-size prefix.
